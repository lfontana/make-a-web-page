Important Concepts

Lesson 0: Thinking Like a Programmer

  • Procedural Thinking
    • - Create perfectly clear and unambiguous instructions for a computer to follow. (Because computers are stupid)
  • Abstract Thinking
    • - Find the similarities between seemingly different things.
  • Systems Thinking
    • - Break a big problem down into smaller, more manageable components
  • Technological Empathy
    • - Understand how computers (& other tech) work/operate
  • Debugging
    • - Systemic process for identifying the cause of a computer program that doesn't work as intended in some fashion
    ----->Collect Evidence
    ----->Generate Theories
    ----->Test Theories
    ----->Resolve the Problem

Lesson 1: Basics of Web and HTML

  • What the web is
    • - Network of servers that communicate with one another, responding to HTTP requests and returning appropriate HTML doc and displays on the users screen.
  • Computers are stupid
    • - Computers interpet instructions literally, always. Very rigid and onforgiving.
  • Programmers cannot remember everything
    • - Google is your friend. The key is knowing the questions to ask.
  • HTML
    • - Hyper Text Markup Language. Makes up the majority of content on the web. Made up of:
    -----> Text content (what you see),
    -----> Markup (what it looks like),
    -----> References to other documents, images, videos, et al and Links (to other pages)
    -----> HTML markup is comprised of tags, content (text) and elements
  • Web Applications
    • - Software that runs in a web browser.
  • Basic HTML vocabluary
    • - Tags and Elements
    -----> An HTML doc is comprised of elements. -----> Elements contain tags and text.

    Lesson 2: Creating a structured document with HTML

  • Developer Tools (in browser)
    • - Box-like model of a web page is made up of HTML elements within elements. These boxes for the structure of a webapge.
    - Div and Span are two elements that can be used to build structure for a web page
    - Every page is made up of boxes. Usually boxes within boxes. Think of a webpage as a grid. Like graph paper.

    Lesson 3: Adding CSS Style to HTML Structure

  • Definitions
    • - CSS: Cascading Style Sheets. This is code written to control the style of HTML elements. Think of it as a "search and replace" tool. You ID a class or a tag and then define a rule for what you want to do with it, visually. The 'cascading' refers to how the rules nest within one another, most specific rule appplies.
    - CSS Inheritance : Properties are applies to a specified element AND its descendeants
    - CSS Selector: Allows you to target specfic HTML elements and apply style to them.
  • 3 Ways to include CSS Styling in a WebPage
    • Write CSS in the head of your HTML
    Link your HTML to a separate CSS file (Preferred)
    - One file = HTML
    - One file = CSS
    - Add head element to top of HTML
    - Add link tag inside the head element
    - Add attributes to link
    Write your style inline with your HTML. (Bad juju; results in duplicate code)
  • Box Model
    • HTML elements are boxes and boxes are comprise of four components:
    a) Content. The text or image at the center of the box.
    b) Padding. The clear area around the content.
    c) Border. Outlines the content+padding.
    d) Margin. Transparent area around the border.
    - Size of an element = Content+padding+border. Does not include margin.
    Getting the size of a box right can be a chore.
    Two techniques can help:
    - Set size as a % rather than hard set pixels
    - Set the box-sizing attribute to border-box for each element
    Broweser Specific Prefixes: Boxes will not render exactly the same in each browser. Browser specific prefizes can make it somewhat agnostic.
    - webkit-box-sizing: border-box;
    - moz-box-sizing: border-box;
    - ms-box-sizing: border-box;
    box-sizing: border-box;
  • Box positioning
    • - Divs are block elements and thus take up the entire width of the screen. - Ading rule display:flex; to CSS wil over ride this behavior and allow divs to appear next to one another. Code, Test & Refine: Basic developer practice of continuous improvement. Do, inspect, adapt.
    - Look for the natural boxes
    - Look for repeated styles and semantic elements - Write HTML - Apply styles (CSS) from biggest to smallest - Inspect and change. Go over HTML and CSS, inspecting results and modifying code until you are satisfied.
